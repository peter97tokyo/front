3. alt 속성을 빈 값으로 두는 경우

장식용 이미지 (Decorative Images):

  • 장식용 이미지는 시각적으로만 의미가 있고, 내용 전달에는 중요하지 않은 이미지입니다. 이런 경우에는 alt 속성을 빈 값으로 두어 스크린 리더가 이를 무시하게 할 수 있습니다.
  • 예를 들어, 디자인 요소로 사용된 구분선이나 배경 이미지 등이 있습니다.

CSS로 삽입된 이미지:

  • CSS 배경 이미지처럼, HTML 콘텐츠와는 관계없이 스타일링을 위해 사용되는 이미지도 alt 속성을 빈 값으로 둘 수 있습니다.

이미지를 설명하는 인접 텍스트가 있는 경우:

  • 이미지 바로 옆에 이미지에 대한 설명이 텍스트로 충분히 제공되는 경우, alt 속성을 빈 값으로 두어 중복 설명을 피할 수 있습니다.

언제 alt 속성을 제공해야 하는가?

정보 전달 이미지:

  • 콘텐츠의 중요한 부분을 설명하는 이미지는 반드시 alt 속성을 통해 대체 텍스트를 제공해야 합니다.

링크된 이미지:

  • 링크된 이미지는 링크 목적을 설명하는 alt 텍스트가 필요합니다.

장폼의 컨트롤 이미지:

  • 입력 폼의 버튼이나 다른 컨트롤로 사용되는 이미지는 그 기능을 설명하는 alt 텍스트가 필요합니다.

4. 웹 접근성 측면에서 outline을 제거하는 것은 중요한 문제입니다.

웹 사이트에서 요소를 클릭했을 때 기본적으로 브라우저는 해당 요소에 포커스 링(focus ring)을 표시합니다. 이 포커스 링은 키보드 사용자, 특히 시각 장애가 있는 사용자에게 현재 포커스가 어디에 있는지 알려주는 중요한 시각적 표시입니다.

그러나 outline을 제거하면 키보드 사용자들이 현재 포커스가 어디 있는지 알 수 없어 접근성이 저하됩니다. 이로 인해 사용자가 웹 페이지를 탐색하는 데 어려움을 겪을 수 있습니다. outline을 없애는 대신, 접근성을 유지하면서 시각적으로 더 보기 좋게 만드는 방법이 있습니다.

문제점

키보드 사용자는 Tab 키를 사용하여 웹 페이지 내의 포커스 가능한 요소들(링크, 버튼, 폼 필드 등)을 순차적으로 이동합니다. 기본 outline이 없으면 다음과 같은 문제점이 발생합니다:

  1. 포커스 가시성 저하: 사용자는 현재 포커스가 어디에 있는지 알 수 없습니다.
  2. 접근성 저하: 웹 페이지가 키보드로 탐색하기 어렵게 됩니다.
  3. 사용자 경험 저하: 특히 시각 장애가 있는 사용자에게 불편을 초래합니다.

잘못된 방법

            
                button:focus {
                    outline: none;
                }

접근성을 유지하면서 스타일링하는 방법

            
                button:focus {
                    outline: 2px solid #005fcc; /* 포커스 시 파란색 테두리 */
                    outline-offset: 2px; /* 포커스 링을 요소 밖으로 약간 이동 */
                }
                
                a:focus {
                    outline: 2px dashed #005fcc; /* 포커스 시 파란색 점선 테두리 */
                    outline-offset: 2px; /* 포커스 링을 요소 밖으로 약간 이동 */
                }

또 다른 방법: box-shadow 사용

            
                button:focus {
                    box-shadow: 0 0 3px 3px rgba(0, 95, 204, 0.5); /* 파란색 그림자 */
                }
                
                a:focus {
                    box-shadow: 0 0 3px 3px rgba(0, 95, 204, 0.5); /* 파란색 그림자 */
                }

웹 접근성 측면에서 alt, role, aria와 같은 보조 속성을 무시하지 말아야 한다는 것은 이러한 속성들이 시각적으로 정보를 얻을 수 없는 사람들에게 필수적인 정보를 제공하기 때문입니다. 이 보조 속성들은 장애가 있는 사용자들이 웹 콘텐츠에 접근하고 이해할 수 있도록 도와줍니다.

alt 속성

alt 속성은 이미지의 대체 텍스트를 제공하는 역할을 합니다. 이미지가 로드되지 않거나 시각 장애가 있는 사용자가 스크린 리더를 사용하는 경우, alt 속성의 내용이 이미지 대신 읽혀집니다.

aria 속성

ARIA(Accessible Rich Internet Applications)는 웹 애플리케이션의 접근성을 높이기 위한 속성들을 제공합니다. ARIA 속성들은 특히 동적 콘텐츠와 사용자 인터페이스 컴포넌트의 접근성을 개선하는 데 사용됩니다.

  • aria-label: 요소에 레이블을 제공하여 스크린 리더가 이를 읽을 수 있도록 합니다.
  • aria-hidden: 요소를 스크린 리더가 무시하도록 합니다.
이 텍스트는 스크린 리더에서 읽히지 않습니다.

결론

이러한 보조 속성들을 무시하지 말아야 하는 이유는 웹 접근성을 향상시키기 위해서입니다. 시각적 정보를 얻을 수 없는 사용자들이 웹 페이지의 콘텐츠를 이해하고 탐색할 수 있도록 돕는 것은 매우 중요합니다.
